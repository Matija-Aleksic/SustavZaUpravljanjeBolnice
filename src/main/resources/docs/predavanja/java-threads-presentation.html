1️⃣ Način 1: extends Thread

Najstariji i najjednostavniji pristup - nasljeđivanje Thread klase

Java 1.0+ Legacy pristup
MojThread.java Java 
// Način 1: Nasljeđivanje Thread klase
public class MojThread extends Thread {
    
    private String imeTaska;
    
    public MojThread(String imeTaska) {
        this.imeTaska = imeTaska;
    }
    
    @Override
    public void run() {
        // Ovaj kod se izvršava u novom threadu
        for (int i = 1; i <= 5; i++) {
            System.out.println(imeTaska + " - korak " + i);
            try {
                Thread.sleep(500); // Pauza 500ms
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
    }
}
Main.java - Pokretanje Java 
public class Main {
    public static void main(String[] args) {
        // Kreiranje thread objekata
        MojThread thread1 = new MojThread("⬇️ Download");
        MojThread thread2 = new MojThread("⬆️ Upload");
        
        // Pokretanje threadova - poziva run() u novoj niti!
        thread1.start();  // NE pozivati run() direktno!
        thread2.start();
        
        System.out.println("🚀 Main thread nastavlja...");
    }
}
⚠️
Zašto je ovo "legacy"?
Java podržava samo jednostruko nasljeđivanje! Ako naslijediš Thread, ne možeš naslijediti drugu klasu. Zato se preporučuje Runnable.

🆚 start() vs run()

Kritična razlika koju moraš razumjeti!

❌
run() - KRIVO!
Poziva metodu run() u ISTOM threadu. Nema paralelnog izvršavanja!
Krivi način Java 
// ❌ OVO JE KRIVO!
thread1.run();  // Izvršava se u main threadu
thread2.run();  // Čeka da thread1.run() završi

// Ispis je SEKVENCIJALAN:
// Download - korak 1
// Download - korak 2
// ... (sve download)
// Upload - korak 1
// ... (sve upload)
✅
start() - ISPRAVNO!
Kreira NOVI thread i poziva run() u njemu. Paralelno izvršavanje!
Ispravan način Java 
// ✅ OVO JE ISPRAVNO!
thread1.start();  // Novi thread za download
thread2.start();  // Novi thread za upload

// Ispis je PARALELAN (izmješan):
// Download - korak 1
// Upload - korak 1
// Download - korak 2
// Upload - korak 2
// ...
💡
Pro tip za intervjue

import java.util.concurrent.*;

public class CryptoPrice {
    public static void main(String[] args) throws Exception {
        ExecutorService executor = Executors.newFixedThreadPool(3);
        
        // Callable koji "dohvaća" cijenu (simulacija API poziva)
        Callable<Double> btcPrice = () -> {
            Thread.sleep(1000);  // Simulacija mrežnog poziva
            return 67500.50;  // Bitcoin cijena u USD
        };
        
        Callable<Double> ethPrice = () -> {
            Thread.sleep(800);
            return 3450.25;   // Ethereum cijena
        };
        
        // Submit vraća Future objekte
        Future<Double> btcFuture = executor.submit(btcPrice);
        Future<Double> ethFuture = executor.submit(ethPrice);
        
        // Dohvati rezultate (blokira dok nisu spremni)
        System.out.println("₿ BTC: $" + btcFuture.get());
        System.out.println("Ξ ETH: $" + ethFuture.get());
        
        executor.shutdown();
    }
}

4️⃣ Način 4: ExecutorService

Thread pool - profesionalni pristup upravljanju threadovima

Java 5+ Standardni pristup

Kreiranje novog threada za svaki task je skupo! ExecutorService koristi pool threadova koji se recikliraju.

🔢
newFixedThreadPool(n)
Pool s fiksnim brojem threadova. Idealno kada znaš koliko paralelnih taskova želiš.
📈
newCachedThreadPool()
Dinamički pool - kreira nove threadove po potrebi, reciklira neaktivne.
1️⃣
newSingleThreadExecutor()
Samo jedan thread - taskovi se izvršavaju sekvencijalno, jedan po jedan.
⏰
newScheduledThreadPool(n)
Za zakazane ili periodične taskove (npr. svaku minutu).
ExecutorService primjer Java 
import java.util.concurrent.*;

public class ThreadPoolDemo {
    public static void main(String[] args) {
        // Kreiraj pool s 4 threada
        ExecutorService executor = Executors.newFixedThreadPool(4);
        
        // Submitaj 10 taskova (4 se izvršavaju paralelno, ostali čekaju)
        for (int i = 1; i <= 10; i++) {
            int taskId = i;
            executor.submit(() -> {
                String thread = Thread.currentThread().getName();
                System.out.println("Task " + taskId + " na " + thread);
                try {
                    Thread.sleep(1000);
                } catch (InterruptedException e) { }
            });
        }
        
        // VAŽNO: Zatvori executor kada završiš!
        executor.shutdown();
        try {
            executor.awaitTermination(1, TimeUnit.MINUTES);
        } catch (InterruptedException e) { }
    }
}

5️⃣ Način 5: Fork/Join Framework

Divide and conquer za velike zadatke

Java 7+ Paralelizam

🔀 Fork/Join koncept

Veliki Task
[1...1000000]
→ FORK →
Subtask 1
[1...500000]
Subtask 2
[500001...1000000]
→ JOIN →
Rezultat
Suma svih
RecursiveTask - Suma velikog niza Java 
import java.util.concurrent.*;

public class SumaTask extends RecursiveTask<Long> {
    private static final int THRESHOLD = 10_000;
    private final long[] numbers;
    private final int start, end;
    
    public SumaTask(long[] numbers, int start, int end) {
        this.numbers = numbers;
        this.start = start;
        this.end = end;
    }
    
    @Override
    protected Long compute() {
        int length = end - start;
        
        // Bazni slučaj - dovoljno mali za direktnu obradu
        if (length <= THRESHOLD) {
            long sum = 0;
            for (int i = start; i < end; i++) {
                sum += numbers[i];
            }
            return sum;
        }
        
        // Podijeli na dva subtaska
        int mid = start + length / 2;
        SumaTask leftTask = new SumaTask(numbers, start, mid);
        SumaTask rightTask = new SumaTask(numbers, mid, end);
        
        leftTask.fork();   // Pokreni lijevi async
        long rightResult = rightTask.compute();  // Desni u ovom threadu
        long leftResult = leftTask.join();       // Čekaj lijevi
        
        return leftResult + rightResult;
    }
}

import java.util.concurrent.*;
import java.util.stream.*;

public class ForkJoinDemo {
    public static void main(String[] args) {
        // Kreiraj veliki niz brojeva
        long[] numbers = LongStream.rangeClosed(1, 10_000_000).toArray();
        
        // Kreiraj ForkJoinPool (ili koristi common pool)
        ForkJoinPool pool = new ForkJoinPool();
        
        // Kreiraj glavni task
        SumaTask task = new SumaTask(numbers, 0, numbers.length);
        
        // Izmjeri vrijeme
        long start = System.currentTimeMillis();
        
        // Pokreni i dohvati rezultat
        long result = pool.invoke(task);
        
        long duration = System.currentTimeMillis() - start;
        
        System.out.println("📊 Suma: " + result);
        System.out.println("⏱️ Vrijeme: " + duration + "ms");
        System.out.println("🔧 Threadovi: " + pool.getPoolSize());
        
        pool.shutdown();
    }
}

6️⃣ Način 6: CompletableFuture

Asinkrono programiranje na steroidima 💪

Java 8+ Moderni async

CompletableFuture omogućuje lančanje operacija, kombiniranje rezultata i elegantno rukovanje greškama - sve asinkrono!

Osnovne operacije Java 
import java.util.concurrent.*;

public class CompletableFutureBasics {
    public static void main(String[] args) {
        // 1. Pokreni async task
        CompletableFuture<String> future = CompletableFuture.supplyAsync(() -> {
            System.out.println("🔄 Dohvaćam podatke...");
            sleep(1000);
            return "Podaci s API-ja";
        });
        
        // 2. Lančaj transformaciju (ne blokira!)
        CompletableFuture<String> processed = future
            .thenApply(data -> data.toUpperCase())
            .thenApply(data -> "✅ " + data);
        
        // 3. Kad je gotovo, izvrši akciju
        processed.thenAccept(result -> 
            System.out.println("Rezultat: " + result)
        );
        
        // Main thread nastavlja...
        System.out.println("⚡ Main thread radi svoje...");
        
        // Čekaj završetak (samo za demo)
        processed.join();
    }
    
    static void sleep(int ms) {
        try { Thread.sleep(ms); } catch (Exception e) {}
    }
}
💡
Kao JavaScript Promises!
Ako znaš JavaScript, CompletableFuture je kao Promise - thenApply() ~ .then(), exceptionally() ~ .catch()

public class UserDashboard {
    public static void main(String[] args) {
        String userId = "user123";
        
        // Tri paralelna API poziva
        CompletableFuture<String> profileFuture = 
            CompletableFuture.supplyAsync(() -> fetchProfile(userId));
            
        CompletableFuture<Integer> ordersFuture = 
            CompletableFuture.supplyAsync(() -> fetchOrderCount(userId));

    static Integer fetchOrderCount(String id) {

🛡️ CompletableFuture - Error Handling

Elegantno rukovanje greškama u async kodu

Error handling metode Java 
public class ErrorHandlingDemo {
    public static void main(String[] args) {
        // 1. exceptionally() - fallback vrijednost
        CompletableFuture<String> future1 = CompletableFuture
            .supplyAsync(() -> {
                if (Math.random() > 0.5) {
                    throw new RuntimeException("💥 API nije dostupan!");
                }
                return "Podaci s API-ja";
            })
            .exceptionally(ex -> {
                System.out.println("⚠️ Greška: " + ex.getMessage());
                return "Fallback podaci";  // Vraća default
            });
        
        // 2. handle() - obradi i uspjeh i grešku
        CompletableFuture<String> future2 = CompletableFuture
            .supplyAsync(() -> riskyOperation())
            .handle((result, ex) -> {
                if (ex != null) {
                    return "❌ Error: " + ex.getMessage();
                }
                return "✅ Success: " + result;
            });
        
        // 3. whenComplete() - reagiraj bez mijenjanja rezultata
        CompletableFuture<String> future3 = CompletableFuture
            .supplyAsync(() -> "Data")
            .whenComplete((result, ex) -> {
                if (ex == null) {
                    System.out.println("📝 Log: Uspješno dohvaćeno");
                } else {
                    System.out.println("📝 Log: Greška - " + ex);
                }
            });
        
        // Dohvati rezultate
        System.out.println(future1.join());
        System.out.println(future2.join());
    }
}
🔄
exceptionally()
Vraća fallback vrijednost ako dođe do greške. Kao catch block.
🎯
handle()
Prima i rezultat i exception - možeš obraditi oba slučaja.
📋
whenComplete()
Side-effect (logging), ne mijenja rezultat pipeline-a.

7️⃣ Način 7: Virtual Threads

Kreiranje virtualnih threadova u Javi 21+

Načini kreiranja Virtual Threadova Java 21+ 
public class VirtualThreadDemo {
    public static void main(String[] args) throws Exception {
        
        // 1️⃣ Thread.startVirtualThread() - najjednostavnije
        Thread vt1 = Thread.startVirtualThread(() -> {
            System.out.println("🌟 Pozdrav iz virtual threada!");
        });
        vt1.join();
        
        // 2️⃣ Thread.ofVirtual() - builder pattern
        Thread vt2 = Thread.ofVirtual()
            .name("moj-virtual-thread")
            .start(() -> {
                System.out.println("🎯 Thread: " + 
                    Thread.currentThread().getName());
            });
        vt2.join();
        
        // 3️⃣ Executors.newVirtualThreadPerTaskExecutor()
        try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
            for (int i = 0; i < 10; i++) {
                int taskNum = i;
                executor.submit(() -> {
                    System.out.println("⚡ Task " + taskNum + 
                        " na " + Thread.currentThread());
                });
            }
        }
        
        // Provjeri je li thread virtualan
        System.out.println("Je li virtualan? " + vt1.isVirtual());
    }
}

Pro tip
Koristi newVirtualThreadPerTaskExecutor() za I/O bound taskove (mrežni pozivi, baza podataka). Za CPU-bound taskove, i dalje koristi platform threadove!

🌐 Virtual Threads - Web Server

Praktični primjer - HTTP server s milijun konekcija

SimpleWebServer.java Java 21+ 
import java.net.*;
import java.io.*;
import java.util.concurrent.*;

public class SimpleWebServer {
    public static void main(String[] args) throws IOException {
        try (ServerSocket server = new ServerSocket(8080)) {
            System.out.println("🚀 Server pokrenut na portu 8080");
            
            // Virtual thread per request!
            try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
                while (true) {
                    Socket client = server.accept();
                    
                    // Svaki request dobiva svoj virtual thread
                    executor.submit(() -> handleRequest(client));
                }
            }
        }
    }
    
    static void handleRequest(Socket client) {
        try (var in = new BufferedReader(new InputStreamReader(client.getInputStream()));
             var out = new PrintWriter(client.getOutputStream(), true)) {
            
            // Simulacija spore operacije (DB query)
            Thread.sleep(100);  // Virtual thread se "parkira", carrier je slobodan!
            
            String response = "HTTP/1.1 200 OK\r\n\r\nHello from Virtual Thread!";
            out.println(response);
            
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

extends Thread 1.0+ Izbjegavaj - legacy ⭐
implements Runnable 1.0+ Jednostavni taskovi bez rezultata ⭐⭐⭐
Callable + Future 5+ Taskovi koji vraćaju rezultat ⭐⭐⭐
ExecutorService 5+ Većina use case-ova, thread pool ⭐⭐⭐⭐
Fork/Join 7+ Veliki rekurzivni problemi, divide & conquer ⭐⭐⭐
CompletableFuture 8+ Async pipelines, kombiniranje rezultata ⭐⭐⭐⭐⭐
Virtual Threads 21+ I/O bound, masivna skalabilnost ⭐⭐⭐⭐⭐
