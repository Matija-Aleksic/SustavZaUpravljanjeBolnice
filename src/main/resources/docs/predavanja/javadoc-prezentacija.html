📚 Što je Javadoc dokumentacija?

Javadoc je sustav za generiranje HTML dokumentacije iz posebno formatiranih komentara u Java kodu.

🎯 Zašto je važan?

  • 📖 Razumljivost: Pomaže drugim programerima (i vama nakon 6 mjeseci) da razumiju vaš kod
  • 🤝 Timski rad: Olakšava kolaboraciju na projektima
  • ⚡ IDE podrška: Moderna razvojna okruženja (IntelliJ, Eclipse, VS Code) automatski prikazuju Javadoc
  • 🏢 Profesionalnost: Standard u industriji - obavezan u većini kompanija
⚠️ Upozorenje: Kod bez dokumentacije = tehnički dug = glavobolja za cijeli tim!

🎓 Primjer: Dokumentirana klasa

/** * Predstavlja studenta na sveučilištu. * * Ova klasa omogućava upravljanje osnovnim podacima o studentu * uključujući ime, prezime, JMBAG i prosjek ocjena. * * @author Vaše Ime * @version 2.0 * @since Java 25 */ public class Student { private String ime; private String prezime; private String jmbag; private double prosjek; /** * Konstruira novog studenta sa zadanim podacima. * * @param ime ime studenta (ne smije biti null) * @param prezime prezime studenta (ne smije biti null) * @param jmbag jedinstveni matični broj (format: 10 znamenki) * @throws IllegalArgumentException ako je parametar null ili JMBAG neispravan */ public Student(String ime, String prezime, String jmbag) { this.ime = ime; this.prezime = prezime; this.jmbag = jmbag; } }

⚙️ Primjer: Dokumentirana metoda

/** * Izračunava konačnu ocjenu studenta na temelju prosjeka. * * Konačna ocjena se određuje prema sljedećem: * <ul> * <li>1.5 - 2.5: Dovoljan (2)</li> * <li>2.5 - 3.5: Dobar (3)</li> * <li>3.5 - 4.5: Vrlo dobar (4)</li> * <li>4.5 - 5.0: Izvrstan (5)</li> * </ul> * * @param prosjek prosjek ocjena studenta (1.0 - 5.0) * @return konačna ocjena kao cijeli broj (2-5) * @throws IllegalArgumentException ako je prosjek izvan raspona * @see #getProsjek() * @since 2.0 */ public int izracunajKonacnuOcjenu(double prosjek) { if (prosjek < 1.5 || prosjek > 5.0) { throw new IllegalArgumentException("Neispravan prosjek"); } return (int) Math.round(prosjek); }

🎨 Kako to izgleda u dokumentaciji?

izracunajKonacnuOcjenu

public int izracunajKonacnuOcjenu(double prosjek)

Izračunava konačnu ocjenu studenta na temelju prosjeka.

Konačna ocjena se određuje prema sljedećem:

  • 1.5 - 2.5: Dovoljan (2)
  • 2.5 - 3.5: Dobar (3)
  • 3.5 - 4.5: Vrlo dobar (4)
  • 4.5 - 5.0: Izvrstan (5)

Parameters:

prosjek - prosjek ocjena studenta (1.0 - 5.0)

Returns:

konačna ocjena kao cijeli broj (2-5)

Throws:

IllegalArgumentException - ako je prosjek izvan raspona

🔗 Javadoc i nasljeđivanje

Java automatski nasljeđuje Javadoc dokumentaciju! 🎉

Automatsko nasljeđivanje u slučajevima:

  • @Override u podklasi: Automatski se dodaje oznaka "Overrides"
  • Implementacija sučelja: Automatski se dodaje oznaka "Specified by"
  • Nasljeđivanje sučelja: Dokumentacija se prenosi s nadsučelja
// U sučelju public interface Osoba { /** * Vraća puno ime osobe. * * @return ime i prezime odvojeni razmakom */ String getPunoIme(); } // U klasi - NE TREBA ponovno pisati Javadoc! public class Student implements Osoba { private String ime; private String prezime; @Override public String getPunoIme() { return ime + " " + prezime; } // Javadoc se automatski naslijeđuje! ✨ }

✨ Zlatno pravilo: Prva rečenica

⚡ Najvažnije pravilo Javadoc dokumentacije!

Prva rečenica MORA biti sažetak koji se može čitati samostalno.

✅ DOBRO

/** * Izračunava prosjek ocjena. * * Metoda uzima u obzir samo * pozitivne ocjene i zaokružuje * rezultat na dvije decimale. */

✓ Prva rečenica je jasna i samostalna

✓ Počinje glagolom

✓ Detalji su odvojeni

❌ LOŠE

/** * Ova metoda se koristi kada * trebamo izračunati prosjek. * Izračunava prosjek ocjena. */

✗ Prva rečenica je nejasna

✗ Ne počinje glagolom

✗ Previše riječi bez smisla

💡 Pravilo: Prva rečenica završava s prvom točkom. Javadoc alat je koristi za pregled!

✍️ Stil pisanja Javadoc dokumentacije

📝 Pravila pisanja

  • Piši u trećem licu ("dohvaća", ne "dohvaćam")
  • Opisi metoda počinju glagolom
  • Koristi <code> za kod
  • Koristi <p> za paragrafe
  • Koristi HTML oznake za formatiranje
💡 HTML u Javadoc: Možeš koristiti osnovne HTML tagove poput <ul>, <ol>, <li>, <b>, <i>, <code>, <pre>

✅ Primjer dobrog stila

/** * Dohvaća listu aktivnih studenata. * <p> * Metoda filtrira studente prema * sljedećim kriterijima: * <ul> * <li>Status: aktivan</li> * <li>Prosjek: > 2.0</li> * </ul> * <p> * Rezultati su sortirani po * prezimenu. * * @return nemodificibilna lista * {@code Student} objekata * @since 2.0 */
❌ Što ne raditi: Ne pišite "metoda()" sa zagradama, već samo "metoda". Javadoc automatski prepoznaje metode!

🚀 Moderne Javadoc značajke u Java 25

1. {@snippet} tag (od Java 18+)

/** * Primjer korištenja: * {@snippet : * Student student = new Student("Marko", "Marić", "0036512345"); * student.setProsjek(4.5); * database.save(student); * } */

2. {@link} i {@linkplain}

/** * Koristi {@link Student} klasu za manipulaciju studentima. * Vidi također {@linkplain DatabaseManager#connect()}. */

3. {@value} za konstante

/** * Maksimalni broj studenata: {@value MAX_STUDENTS} */ public static final int MAX_STUDENTS = 1000;
🎯 Pro tip: Moderne IDE-ove automatski prikazuju Javadoc dok tipkate - iskoristite to!

🏆 Best Practices - Pravila profića

✅ UVIJEK

  • Dokumentiraj sve public metode
  • Dokumentiraj sve parametre
  • Objasni što metoda radi, NE kako
  • Koristi primjere koda
  • Navedi edge case-eve
  • Navedi sve moguće iznimke
  • Ažuriraj dokumentaciju kad mijenjašaš kod!

❌ NIKAD

  • Ne pišite očite stvari
  • Ne kopirajte kod u dokumentaciju
  • Ne ostavljaj zastarjelu dokumentaciju
  • Ne dokumentiraj private metode (osim ako su kompleksne)
  • Ne pišite romane - budi koncizan

💎 Zlatna pravila

1. Dokumentacija koja laže je gora od nedostatka dokumentacije

2. Ako trebaš dokumentirati "zašto" metoda postoji - možda je problem u dizajnu

3. Dobar kod se sam dokumentira, ali Javadoc je i dalje neophodan!

⚠️ Česti problemi i kako ih riješiti

❌ Problem 1: Neažurirana dokumentacija

/** * @param ime ime studenta * @param prezime prezime studenta */ public Student(String ime, String prezime, String jmbag) { // Zaboravio si dodati @param za jmbag! ⚠️ this.ime = ime; this.prezime = prezime; this.jmbag = jmbag; }

Rješenje: Koristi IDE upozorenja i provjere prije commit-a!

❌ Problem 2: Nejasne @param oznake

/** * @param x broj // LOŠE - što je x? */

Rješenje: Uvijek koristi opisna imena i detaljne opise!

/** * @param brojStudenata ukupan broj studenata (min: 1, max: 50) * // DOBRO! ✓ */

🎯 Stvarni primjer: Kompletan projekt

package hr.fer.oop.studenti; import java.util.*; /** * Menadžer za upravljanje studentima na fakultetu. * <p> * Ova klasa pruža funkcionalnosti za: * <ul> * <li>Dodavanje i brisanje studenata</li> * <li>Pretraživanje studenata po različitim kriterijima</li> * <li>Generiranje izvještaja</li> * </ul> * <p> * Primjer korištenja: * {@snippet : * StudentManager manager = new StudentManager(); * Student s = new Student("Ana", "Anić", "0036512345"); * manager.addStudent(s); * } * * @author Vaše Ime * @version 3.0 * @since Java 25 * @see Student */ public class StudentManager { private Map<String, Student> studenti = new HashMap<>(); /** * Dodaje novog studenta u sustav. * <p> * Ako student sa istim JMBAG-om već postoji, * baca se iznimka. * * @param student student za dodavanje (ne smije biti null) * @throws IllegalArgumentException ako je student null * @throws DuplicateStudentException ako student već postoji * @since 1.0 */ public void addStudent(Student student) { if (student == null) { throw new IllegalArgumentException("Student ne smije biti null"); } if (studenti.containsKey(student.getJmbag())) { throw new DuplicateStudentException( "Student sa JMBAG-om već postoji" ); } studenti.put(student.getJmbag(), student); } /** * Dohvaća studenta po JMBAG-u. * * @param jmbag jedinstveni matični broj studenta * @return Student objekt ili null ako ne postoji * @since 1.0 */ public Student getStudent(String jmbag) { return studenti.get(jmbag); } }
