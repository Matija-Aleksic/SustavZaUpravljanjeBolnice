🧬 Nasljeđivanje među klasama

Nasljeđivanje omogućava kreiranje novih klasa koje koriste članove već postojećih klasa

Ključni pojmovi:

  • Nadklasa (superclass) - postojeća klasa
  • Podklasa (subclass) - nova klasa koja nasljeđuje
  • Podklasa dodaje nove značajke i modificira ponašanje nadklase
  • Sve klase u Javi nasljeđuju java.lang.Object
⚠️ Važno: Java podržava samo jednostruko nasljeđivanje - svaka klasa može nasljeđivati samo jednu nadklasu!

Primjer nasljeđivanja

// Nadklasa - općenita klasa
public class BankovniKredit {
    protected double iznos;
    protected double kamatnaStopa;
    protected int rokOtplate;
    
    public double izracunajMjesecnuRatu() {
        // Osnovna formula za izračun rate
        return iznos * (1 + kamatnaStopa) / rokOtplate;
    }
}

// Podklasa - specifičnija implementacija
public class KreditZaAuto extends BankovniKredit {
    private String markaAutomobila;
    private int godinaProizvodnje;
    
    @Override
    public double izracunajMjesecnuRatu() {
        // Poseban izračun za auto kredite
        double osnovnaRata = super.izracunajMjesecnuRatu();
        return osnovnaRata * 0.95; // 5% popusta
    }
}

🔐 Modifikatori pristupa

Modifikator Pristup
public Dostupno svugdje

private Dostupno samo unutar klase

⬆️ Ključna riječ super

Korištenje:

  • super - referenca na nadklasu
  • this - referenca na trenutnu klasu
public class DvodimenzionalniOblik {
    protected double visina;
    protected double sirina;
    
    public DvodimenzionalniOblik(double v, double s) {
        this.visina = v;
        this.sirina = s;
    }
}

public class Pravokutnik extends DvodimenzionalniOblik {
    
    public Pravokutnik(double v, double s) {
        // Poziv konstruktora nadklase
        super(v, s);
    }
    
    public double getVisina() {
        // Pristup varijabli nadklase
        return super.visina;
    }
}

🏗️ Konstruktori u podklasamas

⚠️ Važno pravilo: Prva naredba svakog konstruktora podklase je super() 
public class Zaposlenik {
    private String ime;
    
    public Zaposlenik(String ime) {
        this.ime = ime;
    }
}

public class Profesor extends Zaposlenik {
    private String katedra;
    
    public Profesor(String ime, String katedra) {
        // MORA biti prva naredba!
        super(ime);
        this.katedra = katedra;
    }
}

Konstruktori se pozivaju ulančano - od nadklase prema podklasi

🔄 Nadjačavanje metoda (@Override)

public class DvodimenzionalniOblik {
    private double visina;
    private double sirina;
    
    public double izracunajPovrsinu() {
        return visina * sirina;
    }
}

public class Krug extends DvodimenzionalniOblik {
    private double polumjer;
    
    // Anotacija označava nadjačavanje
    @Override
    public double izracunajPovrsinu() {
        return Math.PI * Math.pow(polumjer, 2);
    }
}
✅ Prednosti @Override: Kompajler provjerava da li pravilno nadjačavate metodu i javlja grešku ako nije!

🔷 Klasa java.lang.Object

Sve klase u Javi nasljeđuju Object - "majka svih klasa"

Metoda Opis
equals() Usporedba objekata (preporučeno nadjačati!)
hashCode() Hash vrijednost objekta (za kolekcije)
toString() String reprezentacija objekta
clone() Kopiranje objekta
wait(), notify() Rad s nitima i sinkronizacija

Nadjačavanje equals() i hashCode()

public class Student {
    private String oib;
    private String ime;
    
    @Override
    public boolean equals(Object obj) {
        if (this == obj) return true;
        if (obj == null || getClass() != obj.getClass()) 
            return false;
        Student student = (Student) obj;
        return Objects.equals(oib, student.oib);
    }
    
    @Override
    public int hashCode() {
        return Objects.hash(oib);
    }
    
    @Override
    public String toString() {
        return "Student{oib='" + oib + "', ime='" + ime + "'}";
    }
}
💡 Pravilo: Ako nadjačate equals(), morate nadjačati i hashCode()!

🎭 Polimorfizam

Polimorfizam = "Mnogo oblika" - ista metoda, različito ponašanje

Prednosti polimorfizma:

  • ✅ Općenito programiranje umjesto specifičnog
  • ✅ Lakše proširivanje sustava
  • ✅ Tipovi se određuju tijekom izvođenja
  • ✅ Fleksibilniji kod
// Polje različitih oblika
Oblik[] oblici = new Oblik[3];
oblici[0] = new Krug(5.0);
oblici[1] = new Pravokutnik(4.0, 6.0);
oblici[2] = new Trokut(3.0, 4.0, 5.0);

// Polimorfizam - različite implementacije
for (Oblik oblik : oblici) {
    System.out.println("Površina: " + oblik.izracunajPovrsinu());
}

🔄 Casting objekata

// Upcasting (implicitno)
Oblik oblik = new Krug(5.0);  // Automatski

// Downcasting (eksplicitno - opasno!)
Krug krug = (Krug) oblik;
krug.setPolumjer(10.0);

// Sigurna provjera prije castinga
if (oblik instanceof Krug) {
    Krug k = (Krug) oblik;
    System.out.println("Polumjer: " + k.getPolumjer());
}

// Java 16+ - Pattern matching
if (oblik instanceof Krug k) {
    // Varijabla 'k' je automatski dostupna!
    System.out.println("Polumjer: " + k.getPolumjer());
}
✨ Java 16+: Pattern matching za instanceof - kraći i čitljiviji kod!

🎨 Apstraktne klase

⚠️ Važno: Iz apstraktnih klasa nije moguće kreirati objekte! 
public abstract class Oblik {
    protected String naziv;
    
    // Obična metoda s implementacijom
    public String getNaziv() {
        return naziv;
    }
    
    // Apstraktna metoda - BEZ implementacije
    public abstract double izracunajPovrsinu();
    public abstract double izracunajOpseg();
}

public class Krug extends Oblik {
    private double polumjer;
    
    // MORA implementirati sve apstraktne metode
    @Override
    public double izracunajPovrsinu() {
        return Math.PI * polumjer * polumjer;
    }
    
    @Override
    public double izracunajOpseg() {
        return 2 * Math.PI * polumjer;
    }
}

Apstraktne klase - Pravila

✅ Ključna pravila:

  • Označava se ključnom riječju abstract
  • Može sadržavati i obične i apstraktne metode
  • Ako ima barem jednu apstraktnu metodu → klasa MORA biti apstraktna
  • Podklase MORAJU implementirati sve apstraktne metode
  • Ako podklasa ne implementira sve → i ona mora biti apstraktna
  • Mogu imati vlastiti konstruktor
💡 Kada koristiti: Kada želite definirati zajedničko ponašanje, ali implementacija ovisi o konkretnom tipu objekta

📋 Definiranje sučelja (Interfaces)

Sučelja definiraju "ugovor" - koje metode klasa mora implementirati

// Definiranje sučelja
public interface Elektricno {
    // Konstantne vrijednosti (public static final)
    int MAX_NAPON = 230;
    
    // Apstraktne metode (public abstract)
    void ukljuci();
    void iskljuci();
    boolean jeUkljuceno();
}

// Implementiranje sučelja
public class ElektricnaGitara extends Gitara 
    implements Elektricno {
    
    private boolean ukljucena;
    
    @Override
    public void ukljuci() {
        ukljucena = true;
        System.out.println("Gitara uključena!");
    }
    
    @Override
    public void iskljuci() {
        ukljucena = false;
    }
    
    @Override
    public boolean jeUkljuceno() {
        return ukljucena;
    }
}

🔗 Višestruka implementacija sučelja

✨ Ključna razlika: Klasa može implementirati više sučelja, ali naslijediti samo jednu klasu! 
public interface Plivajuce {
    void pliva();
}

public interface Letece {
    void leti();
}

public interface Trkace {
    void trci();
}

// Implementira VIŠE sučelja!
public class Patka extends Zivotinja 
    implements Plivajuce, Letece, Trkace {
    
    @Override
    public void pliva() {
        System.out.println("Patka pliva!");
    }
    
    @Override
    public void leti() {
        System.out.println("Patka leti!");
    }
    
    @Override
    public void trci() {
        System.out.println("Patka trči!");
    }
}

✨ Java 8: Default metode u sučeljima

Novost u Java 8: Sučelja mogu imati metode s implementacijom! 
public interface PracenjeVremena {
    
    // Default metoda - ima implementaciju
    default String dohvatiDatumVrijeme() {
        LocalDateTime now = LocalDateTime.now();
        return now.format(
            DateTimeFormatter.ofPattern("dd.MM.yyyy. HH:mm:ss")
        );
    }
    
    // Apstraktna metoda
    void zabiljeziAktivnost(String opis);
}

public class Zadatak implements PracenjeVremena {
    
    // Može koristiti default implementaciju
    public void ispisi() {
        System.out.println("Vrijeme: " + dohvatiDatumVrijeme());
    }
    
    // ILI nadjačati default metodu
    @Override
    public String dohvatiDatumVrijeme() {
        return "Custom format...";
    }
    
    @Override
    public void zabiljeziAktivnost(String opis) {
        // Implementacija...
    }
}

Java 8: Statičke metode u sučeljima

public interface Provjera {
    
    // Statička metoda u sučelju
    static boolean provjeriPrazanString(String tekst) {
        return tekst == null || tekst.isEmpty();
    }
    
    static boolean provjeriEmail(String email) {
        return email != null && email.contains("@");
    }
}

// Korištenje - direktno iz sučelja!
public class Main {
    public static void main(String[] args) {
        String ime = "";
        
        // Poziv statičke metode iz sučelja
        if (Provjera.provjeriPrazanString(ime)) {
            System.out.println("Ime je prazno!");
        }
        
        String email = "student@foi.hr";
        if (Provjera.provjeriEmail(email)) {
            System.out.println("Email je valjan!");
        }
    }
}

🚀 Java 8: Lambda izrazi

Lambda izrazi = skraćena sintaksa za anonimne metode

Stari način

// Anonimna klasa
Comparator comp = 
  new Comparator() {
    @Override
    public int compare(String s1, 
                       String s2) {
        return s1.length() - 
               s2.length();
    }
};

Lambda način

// Lambda izraz
Comparator comp = 
    (s1, s2) -> 
        s1.length() - s2.length();

// Još kraće!
Comparator comp2 = 
    Comparator.comparingInt(
        String::length
    );
Sintaksa: (parametri) -> { tijelo }

Funkcionalna sučelja i Lambda

// Funkcionalno sučelje - samo JEDNA apstraktna metoda
@FunctionalInterface
public interface Kalkulator {
    int izracunaj(int a, int b);
}

public class Main {
    public static void main(String[] args) {
        // Lambda izrazi za različite operacije
        Kalkulator zbroj = (a, b) -> a + b;
        Kalkulator razlika = (a, b) -> a - b;
        Kalkulator umnozak = (a, b) -> a * b;
        Kalkulator dijeljenje = (a, b) -> a / b;
        
        System.out.println("Zbroj: " + zbroj.izracunaj(10, 5));
        System.out.println("Razlika: " + razlika.izracunaj(10, 5));
        System.out.println("Umnožak: " + umnozak.izracunaj(10, 5));
        
        // Primjena na liste
        List brojevi = Arrays.asList(1, 2, 3, 4, 5);
        brojevi.forEach(broj -> System.out.println(broj * 2));
    }
}

✨ Java 9: Privatne metode u sučeljima

public interface Sucelje {
    
    // Privatna metoda - može koristiti samo interno
    private void test() {
        System.out.println("Privatna metoda");
    }
    
    // Statička metoda
    public static void test2() {
        System.out.println("Javna statička metoda");
    }
    
    // Default metoda koja poziva privatnu
    public default void test3() {
        test();  // Poziv privatne metode
        System.out.println("Default metoda");
    }
}
💡 Svrha: Privatne metode omogućavaju dijeljenje koda među default metodama bez da ih izlažu van sučelja

🎯 Java 14: Records

Records = kompaktni, nepromjenjivi (immutable) nositelji podataka

Stari način

public class Person {
    private final String name;
    private final String address;
    
    public Person(String name, 
                  String address) {
        this.name = name;
        this.address = address;
    }
    
    public String getName() {
        return name;
    }
    
    public String getAddress() {
        return address;
    }
    
    // + equals, hashCode, 
    // toString...
}

Records način

// Automatski generira:
// - konstruktor
// - getters
// - equals()
// - hashCode()
// - toString()
public record Person(
    String name, 
    String address
) {}

// Korištenje
Person p = new Person(
    "Ana", 
    "Zagreb"
);
System.out.println(
    p.name()
);
✨ Prednosti: Manje koda, thread-safe, savršeno za DTO objekte!

Records - Dodatne mogućnosti

public record Student(
    String oib, 
    String ime, 
    String prezime,
    double prosjek
) {
    // Kompaktni konstruktor - validacija
    public Student {
        if (oib == null || oib.length() != 11) {
            throw new IllegalArgumentException("Neispravan OIB!");
        }
        if (prosjek < 1.0 || prosjek > 5.0) {
            throw new IllegalArgumentException("Prosjek mora biti 1-5!");
        }
    }
    
    // Dodatne metode
    public String punoIme() {
        return ime + " " + prezime;
    }
    
    public boolean jeDobitan() {
        return prosjek >= 4.0;
    }
    
    // Statičke metode
    public static Student kreirajDefault() {
        return new Student("00000000000", "N/A", "N/A", 2.0);
    }
}

🔐 Java 17: Sealed klase i sučelja

Sealed klase omogućavaju kontrolu koje klase mogu nasljeđivati/implementirati 
// Sealed sučelje - SAMO Car i Truck mogu implementirati!
public sealed interface Service 
    permits Car, Truck {
    
    int getMaxServiceIntervalInMonths();
    
    default int getMaxDistanceBetweenServicesInKilometers() {
        return 100000;
    }
}

// Sealed apstraktna klasa
public abstract sealed class Vehicle 
    permits Car, Truck {
    
    protected final String registrationNumber;
    
    public Vehicle(String registrationNumber) {
        this.registrationNumber = registrationNumber;
    }
    
    public String getRegistrationNumber() {
        return registrationNumber;
    }
}

Sealed klase - Implementacija

Final implementacija

// FINAL - ne može se dalje
// nasljeđivati
public final class Truck 
    extends Vehicle 
    implements Service {
    
    private final int loadCapacity;
    
    public Truck(int load, 
                 String regNum) {
        super(regNum);
        this.loadCapacity = load;
    }
    
    @Override
    public int 
      getMaxServiceIntervalInMonths() {
        return 18;
    }
}

Non-sealed implementacija

// NON-SEALED - može se dalje
// nasljeđivati
public non-sealed class Car 
    extends Vehicle 
    implements Service {
    
    private final int numberOfSeats;
    
    public Car(int seats, 
               String regNum) {
        super(regNum);
        this.numberOfSeats = seats;
    }
    
    @Override
    public int 
      getMaxServiceIntervalInMonths() {
        return 12;
    }
}

📊 Usporedba OOP koncepata

Koncept Kada koristiti Primjer
Apstraktna klasa Zajednička implementacija + apstraktne metode Oblik, Zivotinja
Interface Ugovor bez implementacije, višestruka impl. Comparable, Serializable

✅ OOP Best Practices

DO ✅

  • Koristiti @Override anotaciju
  • Nadjačati equals() i hashCode() zajedno
  • Favorizirati kompoziciju nad nasljeđivanjem
  • Koristiti sučelja za fleksibilnost
  • Records za DTO objekte
  • Sealed klase za kontroliranu hijerarhiju

DON'T ❌

  • Duboke hijerarhije nasljeđivanja
  • Pretjerano korištenje nasljeđivanja
  • Zaboraviti super() u konstruktoru
  • Casting bez instanceof provjere
  • Javne promjenjive varijable u klasama
  • Zanemariti princip SRP
