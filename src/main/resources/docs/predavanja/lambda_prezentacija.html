🤔 Što su Lambda izrazi?

Definicija

Lambda izraz je anonimna funkcija koja omogućava pisanje kraćeg, čitljivijeg koda. Uvedeni su u Java 8 i revolucionirali su način programiranja.

💭 Razmisli ovako:

🎵 Spotify Playlist

Umjesto da ručno odabireš svaku pjesmu, samo kažeš: "Pusti sve rock pjesme iz 2023." - Lambda je ta jednostavna naredba!

📸 Instagram Filter

Ne trebaš znati kako filter radi iznutra, samo ga primiješ na sliku jednim klikom. Lambda je kao taj filter za kod!

🎯 Zašto koristiti Lambda izraze?

❌ PRIJE (tradicionalni način)
Sortiranje liste - tradicionalno
// TikTok korisnici po broju followera
List<User> users = getTikTokUsers();

Collections.sort(users, new Comparator<User>() {
    @Override
    public int compare(User u1, User u2) {
        return Integer.compare(
            u1.getFollowers(), 
            u2.getFollowers()
        );
    }
});

// 😓 9 linija koda za jednostavnu operaciju!
✅ S LAMBDA IZRAZOM
Sortiranje liste - lambda
// TikTok korisnici po broju followera
List<User> users = getTikTokUsers();

users.sort((u1, u2) -> 
    Integer.compare(u1.getFollowers(), u2.getFollowers())
);

// 🎉 3 linije - isti rezultat!

// Ili još kraće s method reference:
users.sort(Comparator.comparingInt(User::getFollowers));
💡 Pro Tip
Lambda izrazi čine kod:
  • 📝 Kraćim - manje koda za pisanje i održavanje
  • 👀 Čitljivijim - fokus na ono što radiš, ne kako
  • 🚀 Modernijim - funkcionalni stil programiranja

📝 Sintaksa Lambda izraza

Opća sintaksa

(parametri) -> { tijelo lambda izraza }

→ Arrow operator povezuje parametre s tijelom funkcije

📖 Različiti formati lambda izraza:

1. Bez parametara
// Jednostavna lambda bez parametara
() -> System.out.println("Hello from Lambda! 👋");

// Ili s više linija koda
() -> {
    System.out.println("Pokretanje Stream aplikacije...");
    streamService.start();
    return "Streaming started!";
};
2. Jedan parametar
// Zagrade oko parametra su opcijske
name -> System.out.println("Hi " + name);

// Ili s eksplicitnim tipom (opcijski)
(String name) -> System.out.println("Hi " + name);

// YouTube primjer - subscribe notifikacija
channel -> sendNotification("New video on " + channel);
3. Više parametara
// Zagrade su obavezne za više parametara
(x, y) -> x + y;

// Discord primjer - slanje DM poruke
(user, message) -> user.sendDirectMessage(message);

// S eksplicitnim tipovima
(Integer followers, Integer likes) -> followers + likes;

🔌 Funkcionalna sučelja (Functional Interfaces)

Funkcionalno sučelje je sučelje s točno jednom apstraktnom metodom. Lambda izrazi mogu se koristiti samo s funkcionalnim sučeljima!

Vlastito funkcionalno sučelje
// @FunctionalInterface anotacija osigurava da imamo samo jednu metodu
@FunctionalInterface
interface ContentModerator {
    boolean isAppropriate(String content);
}

// Korištenje s lambdom
ContentModerator moderator = content -> 
    !content.contains("spam") && 
    !content.contains("inappropriate");

// Provjera sadržaja
boolean canPost = moderator.isAppropriate("Check out my new video!");
System.out.println("Can post: " + canPost); // true
⚡ Quick Fact
Java ima više ugrađenih funkcionalnih sučelja u java.util.function paketu. Ne moraš uvijek praviti svoje!

🎁 Ugrađena funkcionalna sučelja

Predicate<T>

Prima argument, vraća boolean

// Filter za Instagram story
Predicate<Story> isActive = 
    story -> story.getHoursOld() < 24;

// Provjera
if (isActive.test(myStory)) {
    showStory(myStory);
}
Function<T,R>

Prima argument tipa T, vraća rezultat tipa R

// Username formatter
Function<String, String> addAtSign = 
    name -> "@" + name.toLowerCase();

String handle = addAtSign.apply("JohnDoe");
// Result: "@johndoe"
Consumer<T>

Prima argument, ne vraća ništa

// Slanje push notifikacije
Consumer<User> sendNotif = 
    user -> notificationService
        .send(user.getId(), "New message!");

users.forEach(sendNotif);
Supplier<T>

Ne prima argumente, vraća rezultat

// Random quote generator
Supplier<String> randomQuote = () -> {
    String[] quotes = {
        "Stay hungry",
        "Think different",
        "Just do it"
    };
    return quotes[new Random()
        .nextInt(quotes.length)];
};

String quote = randomQuote.get();

🔗 Method References

Method reference je još kraći način pisanja lambda izraza kada lambda samo poziva postojeću metodu.

Sintaksa: Class::method

Lambda izraz
// Lambda koji poziva metodu
users.forEach(user -> 
    System.out.println(user)
);
Method Reference
// Ista stvar, kraće
users.forEach(System.out::println);

🎯 Vrste Method References:

1. Reference na statičku metodu
// Parsiranje brojeva iz stringa
List<String> numbers = List.of("1", "2", "3");

// Lambda:
numbers.stream().map(s -> Integer.parseInt(s));

// Method reference:
numbers.stream().map(Integer::parseInt);
2. Reference na instance metodu objekta
// Filtriranje praznih stringova
List<String> hashtags = List.of("#java", "", "#coding");

// Lambda:
hashtags.stream().filter(s -> !s.isEmpty());

// Method reference:
hashtags.stream().filter(String::isEmpty).map(s -> !s);
3. Reference na konstruktor
// Kreiranje User objekata
List<String> usernames = List.of("alice", "bob", "charlie");

// Lambda:
usernames.stream().map(name -> new User(name));

// Method reference:
usernames.stream().map(User::new);

🌊 Stream API + Lambda = 🔥

Stream API omogućava funkcionalno procesiranje kolekcija. Lambda izrazi čine Stream API moćnim i lakim za korištenje!

💡 Razmisli o Stream-u kao o "conveyor belt-u"

Podatke stavljaš na "traku" (stream), primjenjuješ operacije (filter, map), i na kraju dobiješ rezultat (collect, count, etc.)

Instagram primjer - najpopularniji postovi
// Lista Instagram postova
List<Post> posts = getInstagramPosts();

// Zadatak: Nađi top 5 postova s najviše lajkova iz 2024.
List<Post> topPosts = posts.stream()
    .filter(post -> post.getYear() == 2024)           // 1. Filtriraj po godini
    .filter(post -> post.getLikes() > 10000)           // 2. Samo popularne
    .sorted(Comparator.comparingInt(Post::getLikes)
            .reversed())                                // 3. Sortiraj silazno
    .limit(5)                                          // 4. Uzmi top 5
    .collect(Collectors.toList());                     // 5. Sakupi u listu

// 🎉 Sve u 6 linija čitljivog koda!
🚀 Stream operacije
  • filter() - filtrira elemente (vraća boolean)
  • map() - transformira elemente (vraća novi tip)
  • sorted() - sortira elemente
  • limit() - ograničava broj elemenata
  • collect() - sakuplja rezultate u kolekciju

List<Song> playlist = songs.stream()

🔄 map() - Transformacije podataka

map() transformira svaki element u stream-u. Prima jedan tip, vraća drugi tip.

TikTok primjer - formatiranje usernames
// Od liste imena do liste @usernames
List<String> names = List.of("Ana", "Marko", "Ivana");

List<String> usernames = names.stream()
    .map(name -> "@" + name.toLowerCase())
    .collect(Collectors.toList());

// Rezultat: ["@ana", "@marko", "@ivana"]
YouTube primjer - ekstrakcija podataka
// Od Video objekata do naslova videa
List<Video> videos = getVideos();

List<String> titles = videos.stream()
    .map(Video::getTitle)
    .collect(Collectors.toList());

// Ukupno pregleda svih videa
long totalViews = videos.stream()
    .map(Video::getViews)
    .reduce(0L, Long::sum);

// Ili kraće:
long totalViews2 = videos.stream()
    .mapToLong(Video::getViews)
    .sum();
Netflix primjer - transformacija Movie -> DTO
// Pretvori Movie objekte u jednostavne DTO-ove
List<MovieDTO> movieCards = movies.stream()
    .map(movie -> new MovieDTO(
        movie.getTitle(),
        movie.getThumbnail(),
        movie.getRating()
    ))
    .collect(Collectors.toList());
💡 Kada koristiti map()?
Koristi map() kada trebaš:
  • Pretvoriti objekte iz jednog tipa u drugi
  • Ekstraktirati određeno polje iz objekta
  • Primijeniti funkciju na svaki element

🗺️ flatMap() - "Ravnanje" struktura

flatMap() se koristi kada imaš "kolekciju kolekcija" i želiš je "spljoštiti" u jednu kolekciju.

map() - bez ravnanja
// Svaki korisnik ima listu prijatelja
List<User> users = getUsers();

// map() vraća Stream<List<User>>
Stream<List<User>> friendLists = 
    users.stream()
        .map(User::getFriends);

// 😕 Lista listi, ne možemo lako raditi
flatMap() - s ravnanjem
// Svaki korisnik ima listu prijatelja
List<User> users = getUsers();

// flatMap() vraća Stream<User>
List<User> allFriends = 
    users.stream()
        .flatMap(user -> user.getFriends().stream())
        .collect(Collectors.toList());

// 🎉 Jedna ravna lista svih prijatelja!
Snapchat primjer - svi hashtagovi
// Svaki story ima više hashtagova
List<Story> stories = getStories();

// Izvuci sve hashtagove iz svih storyja
List<String> allHashtags = stories.stream()
    .flatMap(story -> story.getHashtags().stream())
    .distinct()  // Ukloni duplikate
    .collect(Collectors.toList());

// Rezultat: [#summer, #fun, #travel, #food, ...]
Discord primjer - svi članovi servera
// Svaki server (guild) ima više kanala, svaki kanal više članova
List<Guild> guilds = getGuilds();

Set<User> allMembers = guilds.stream()
    .flatMap(guild -> guild.getChannels().stream())  // Stream kanala
    .flatMap(channel -> channel.getMembers().stream()) // Stream članova
    .collect(Collectors.toSet());  // Set automatski uklanja duplikate

🎯 Collectors - Sakupljanje rezultata

Collectors omogućavaju pretvaranje stream-a u različite strukture podataka.

toList() / toSet()
// U listu
List<String> list = stream
    .collect(Collectors.toList());

// U set (bez duplikata)
Set<String> set = stream
    .collect(Collectors.toSet());
joining()
// Spoji stringove
String hashtags = tags.stream()
    .collect(Collectors.joining(", "));

// Result: "java, coding, tech"
groupingBy()
// Grupiraj po žanru
Map<String, List<Song>> byGenre = 
    songs.stream()
        .collect(Collectors
            .groupingBy(Song::getGenre));
partitioningBy()
// Podijeli u dvije grupe (true/false)
Map<Boolean, List<User>> byStatus = 
    users.stream()
        .collect(Collectors
            .partitioningBy(User::isActive));
Instagram Analytics primjer
// Broj postova po danu u tjednu
Map<DayOfWeek, Long> postsByDay = posts.stream()
    .collect(Collectors.groupingBy(
        post -> post.getDate().getDayOfWeek(),
        Collectors.counting()
    ));

// Prosječan broj lajkova po tipu posta
Map<PostType, Double> avgLikesByType = posts.stream()
    .collect(Collectors.groupingBy(
        Post::getType,
        Collectors.averagingInt(Post::getLikes)
    ));

⚠️ Česte greške s Lambda izrazima

❌ GREŠKA #1: Mutiranje vanjskih varijabli

Loše - Side effects
int sum = 0; // Outer variable
numbers.forEach(n -> {
    sum += n; // ❌ ERROR: Variable used in lambda
              // must be final or effectively final
});
Dobro - Funkcionalni pristup
// Koristi reduce() za agregaciju
int sum = numbers.stream()
    .reduce(0, Integer::sum);

// Ili sum() za primitive
int sum2 = numbers.stream()
    .mapToInt(Integer::intValue)
    .sum();

❌ GREŠKA #2: Presložene lambde

Loše - Nečitljivo
// 😵 Previše logike u lambdi
users.stream()
    .filter(u -> {
        if (u.getAge() > 18) {
            if (u.isActive()) {
                if (u.getFollowers() > 1000) {
                    return true;
                }
            }
        }
        return false;
    });
Dobro - Jasno i čisto
// 😊 Izdvoji u zasebnu metodu
users.stream()
    .filter(this::isInfluencer);

private boolean isInfluencer(User u) {
    return u.getAge() > 18 
        && u.isActive() 
        && u.getFollowers() > 1000;
}

❌ GREŠKA #3: Zanemarivanje null vrijednosti

// ❌ Može baciti NullPointerException
users.stream()
    .map(User::getName)  // getName() može vratiti null!
    .map(String::toUpperCase)  // 💥 NPE!
    .collect(Collectors.toList());

// ✅ Sigurniji pristup
users.stream()
    .map(User::getName)
    .filter(Objects::nonNull)  // Filtriraj null-ove
    .map(String::toUpperCase)
    .collect(Collectors.toList());

// ✅ Ili koristi Optional
users.stream()
    .map(User::getNameOptional)
    .flatMap(Optional::stream)  // Java 9+
    .map(String::toUpperCase)
    .collect(Collectors.toList());

✨ Best Practices

1️⃣ Koristi method reference kada je moguće
// Dobro ✅
list.forEach(System.out::println);

// Meh, može bolje
list.forEach(x -> System.out.println(x));
2️⃣ Izbjegavaj side effects

Lambda izrazi trebaju biti pure functions - bez mutiranja stanja!

// Dobro ✅
return users.stream()
    .filter(User::isActive)
    .collect(Collectors.toList());

// Loše ❌
List<User> result = new ArrayList<>();
users.forEach(u -> {
    if (u.isActive()) {
        result.add(u); // Side effect!
    }
});
3️⃣ Lambda ili običan loop?

Koristi lambdu kada operacija treba biti deklarativna.

// Lambda - za transformacije ✅
names.stream()
    .map(String::toUpperCase)
    .collect(Collectors.toList());

// Loop - za kompleksnu logiku ✅
for (User user : users) {
    if (complexCondition(user)) {
        doMultipleThings(user);
        updateDatabase(user);
    }
}
4️⃣ Ime lambda parametara

Koristi smislena imena, ne generička x, y.

// Dobro ✅
users.stream()
    .filter(user -> user.getAge() > 18)
    .map(user -> user.getName())

// Loše ❌
users.stream()
    .filter(u -> u.getAge() > 18)
    .map(u -> u.getName())

⚡ Performance Savjeti

1. Parallel Streams - Pažljivo!

// Parallel stream za velike kolekcije
List<Integer> numbers = IntStream.range(1, 1_000_000)
    .boxed()
    .collect(Collectors.toList());

// Sequential (obični stream)
long sum1 = numbers.stream()
    .mapToLong(Integer::longValue)
    .sum();

// Parallel - koristi više CPU cores
long sum2 = numbers.parallelStream()
    .mapToLong(Integer::longValue)
    .sum();

// ⚠️ PAZI: Parallel je brži samo za:
// - Velike količine podataka (>10,000 elemenata)
// - CPU-intensive operacije
// - Bez side effects i shared state!

2. Lazy Evaluation

Stream operacije su lazy - izvršavaju se tek kad pozoveš terminal operaciju!

// Ništa se NE izvršava ovdje!
Stream<User> stream = users.stream()
    .filter(user -> {
        System.out.println("Filtering " + user.getName());
        return user.isActive();
    })
    .map(user -> {
        System.out.println("Mapping " + user.getName());
        return user.getName();
    });

// Izvršava se tek kad pozoveš terminal operaciju:
List<String> names = stream.collect(Collectors.toList());

// 💡 Ovo je odlično za performanse - ne procesira sve podatke
// ako koristiš npr. findFirst() ili limit()!

3. Short-circuiting operacije

// anyMatch() - prestaje čim nađe match ✅
boolean hasActiveUsers = users.stream()
    .anyMatch(User::isActive);  // Zaustavlja se na prvom true

// findFirst() - vraća odmah prvog ✅
Optional<User> admin = users.stream()
    .filter(User::isAdmin)
    .findFirst();  // Ne procesira ostale elemente

// limit() - ograničava broj elemenata ✅
List<User> first10 = users.stream()
    .limit(10)
    .collect(Collectors.toList());  // Procesira samo 10
💡 Golden Rule
Nemoj prerano optimizirati! Prvo napiši čitljiv kod, pa optimiziraj ako je potrebno. Stream API je uglavnom dovoljno brz za većinu slučajeva.

🆕 Što je novo u Java 25?

1. Pattern Matching u lambdama (Preview)

// Tradicionalno
users.stream()
    .filter(user -> user instanceof PremiumUser)
    .map(user -> (PremiumUser) user)
    .forEach(premium -> System.out.println(premium.getBenefits()));

// Java 25 - Pattern matching u lambdama
users.stream()
    .filter(PremiumUser premium -> premium.getSubscription().isActive())
    .forEach(premium -> System.out.println(premium.getBenefits()));

2. Enhanced Stream API

// Nove metode za lakšu obradu
List<Post> posts = getPosts();

// mapMulti() - alternative za flatMap
posts.stream()
    .mapMulti((post, consumer) -> {
        post.getTags().forEach(consumer);
    })
    .distinct()
    .forEach(System.out::println);

// takeWhile() i dropWhile() za ordered streams
List<Integer> numbers = List.of(1, 2, 3, 4, 5, 6);

numbers.stream()
    .takeWhile(n -> n < 4)  // [1, 2, 3]
    .forEach(System.out::println);

numbers.stream()
    .dropWhile(n -> n < 4)  // [4, 5, 6]
    .forEach(System.out::println);

3. Record Patterns u lambdama

// Record definicija
record User(String name, int age, String email) {}

// Dekonstrukcija u lambdi
List<User> users = getUsers();

users.stream()
    .map(user -> switch (user) {
        case User(var name, var age, var email) 
            when age > 18 -> "Adult: " + name;
        case User(var name, var age, var email) 
            -> "Minor: " + name;
    })
    .forEach(System.out::println);

🎯 Praktični projekt: Social Media Analytics

Zadatak: Analiza Instagram postova

Kreiraj sustav za analizu popularnosti postova koristeći Stream API i lambde.

Model klase
record Post(
    String id,
    String author,
    String content,
    int likes,
    int comments,
    int shares,
    LocalDateTime postedAt,
    List<String> hashtags
) {
    // Engagement rate calculation
    public double engagementRate() {
        return (likes + comments * 2 + shares * 3) / 100.0;
    }
}
Analytics klasa
public class InstagramAnalytics {
    
    // 1. Top 10 postova po engagement rate-u
    public List<Post> getTopEngagingPosts(List<Post> posts) {
        return posts.stream()
            .sorted(Comparator.comparingDouble(Post::engagementRate).reversed())
            .limit(10)
            .collect(Collectors.toList());
    }
    
    // 2. Najčešći hashtagovi
    public Map<String, Long> getTopHashtags(List<Post> posts, int limit) {
        return posts.stream()
            .flatMap(post -> post.hashtags().stream())
            .collect(Collectors.groupingBy(
                tag -> tag,
                Collectors.counting()
            ))
            .entrySet().stream()
            .sorted(Map.Entry.<String, Long>comparingByValue().reversed())
            .limit(limit)
            .collect(Collectors.toMap(
                Map.Entry::getKey,
                Map.Entry::getValue,
                (e1, e2) -> e1,
                LinkedHashMap::new
            ));
    }
    
    // 3. Autori s prosječno najviše lajkova
    public Map<String, Double> getTopAuthors(List<Post> posts) {
        return posts.stream()
            .collect(Collectors.groupingBy(
                Post::author,
                Collectors.averagingInt(Post::likes)
            ))
            .entrySet().stream()
            .sorted(Map.Entry.<String, Double>comparingByValue().reversed())
            .limit(5)
            .collect(Collectors.toMap(
                Map.Entry::getKey,
                Map.Entry::getValue,
                (e1, e2) -> e1,
                LinkedHashMap::new
            ));
    }
    
    // 4. Postovi iz zadnjih 24 sata
    public List<Post> getRecentPosts(List<Post> posts) {
        LocalDateTime yesterday = LocalDateTime.now().minusHours(24);
        return posts.stream()
            .filter(post -> post.postedAt().isAfter(yesterday))
            .sorted(Comparator.comparing(Post::postedAt).reversed())
            .collect(Collectors.toList());
    }
}

🐛 Debugging Lambda izraza

Tipovi grešaka i kako ih riješiti

1. Debug sa peek()
// peek() omogućava inspekciju elemenata bez mijenjanja stream-a
List<String> result = users.stream()
    .filter(User::isActive)
    .peek(user -> System.out.println("After filter: " + user))
    .map(User::getName)
    .peek(name -> System.out.println("After map: " + name))
    .collect(Collectors.toList());

// Output:
// After filter: User[name=Ana, active=true]
// After map: Ana
// After filter: User[name=Marko, active=true]
// After map: Marko
2. Breakpoints u IntelliJ IDEA
// Postavi breakpoint na liniju s lambdom
users.stream()
    .filter(user -> user.getAge() > 18)  // ← Breakpoint ovdje
    .collect(Collectors.toList());

// IntelliJ će stati na svakom pozivu lambde!
// Možeš vidjeti vrijednosti user objekta
3. Izvuci lambdu u zasebnu metodu
// Loše - teško za debug
users.stream()
    .filter(u -> u.getAge() > 18 && u.isActive() && u.hasEmail())
    .collect(Collectors.toList());

// Dobro - lako za debug
users.stream()
    .filter(this::isEligible)
    .collect(Collectors.toList());

private boolean isEligible(User user) {
    // Ovdje možeš staviti breakpoint i vidjeti sve!
    boolean adultAge = user.getAge() > 18;
    boolean active = user.isActive();
    boolean hasEmail = user.hasEmail();
    return adultAge && active && hasEmail;
}
