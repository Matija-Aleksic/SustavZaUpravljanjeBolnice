Uvod u rad s datotekama

  • Datoteke se koriste za trajnije pohranjivanje podataka ili dohvat informacija iz njih koje su postojale i prije pokretanja programa
  • Postoje dvije vrste datoteka: tekstualne i binarne
  • Iz perspektive Java programa svaka datoteka može se razmatrati kao niz bajtova (engl. stream of bytes)
  • Svaka datoteka na kraju ima "end-of-file" oznaku kojom se detektira kraj datoteke nakon čega je potrebno zaustaviti proces čitanja njenog sadržaja
  • U datoteke je moguće spremati i cijele objekte postupkom koji se naziva serijalizacija, a čitanje podataka o objektima se naziva deserijalizacija
💡 Zanimljivost za Z generaciju
Slično kao što Instagram sprema vaše storije ili TikTok čuva vaše draftove - datoteke omogućuju programima da "zapamte" stvari i nakon što ih ugasite!

Vrste tokova podataka u Javi

  • Tokovi podataka pomoću kojih se dohvaćaju i zapisuju podaci u datoteke mogu biti temeljeni na:

Binarni format

  • Ulazni i izlazni tok predstavljeni bajtovima
  • char ima 2 bajta, int 4 bajta, double 8 bajtova itd.
  • Koristi se kod binarnih datoteka

Znakovni format

  • Ulazni i izlazni tok predstavljen nizom znakova
  • Svaki znak ima 2 bajta
  • Koristi se kod tekstualnih datoteka
  • Prilikom otvaranja datoteka potrebno je kreirati pripadajući objekt čiji konstruktor komunicira s operacijskim sustavom
  • U Javi postoje tri standardna toka podataka:
    • System.in - za učitavanje podataka s tipkovnice
    • System.out - za ispis podataka na zaslon
    • System.err - za ispis podataka o pogreškama na zaslon

Klasa java.io.File

  • Predstavlja element datotečnog sustava (može biti datoteka ili mapa)
  • Sadrži putanju, ime i veličinu datoteke, ali ne i sam sadržaj
  • Može kreirati datoteku na sljedeće načine:
// Kreiranje File objekata
File wf1 = new File("moj.htm");
File wf2 = new File("java\\html\\IO.html");
File wf3 = new File("D:\\java\\25.txt");

// Provjera postojanja i tipa
if (wf1.exists()) {
    System.out.println("Datoteka postoji!");
}

if (wf1.isFile()) {
    System.out.println("To je datoteka");
} else if (wf1.isDirectory()) {
    System.out.println("To je mapa");
}

// Dohvat informacija
System.out.println("Naziv: " + wf1.getName());
System.out.println("Putanja: " + wf1.getPath());
System.out.println("Veličina: " + wf1.length() + " bajtova");
  • Sadrži niz korisnih metoda: exists(), isFile(), isDirectory(), getName(), getPath() itd.

Sučelja i klase iz paketa java.nio.file

Omogućavaju dohvat podataka iz datoteka pomoću sljedećih sučelja i klasa (uvedeni od Jave SE 6):

Sučelje/Klasa Opis
Path (sučelje) Objekti predstavljaju lokaciju datoteka ili mape, ali ne omogućavaju čitanje sadržaja
Paths (klasa) Sadrži statičke metode za dohvat Path objekata koji predstavljaju datoteku ili mapu
Files (klasa) Sadrži statičke metode za rad s datotekama: kopiranje, kreiranje, brisanje, čitanje sadržaja itd.
DirectoryStream (sučelje) Omogućava iteraciju kroz sadržaj mape
🎯 Moderna praksa
NIO.2 API (java.nio.file) je moderniji i preporučeniji način rada s datotekama u odnosu na stari java.io.File!

Klasa Formatter

  • Služi za oblikovanje teksta koji se može zapisivati u datoteku
  • Radi po principu korištenja metode printf
import java.util.Formatter;
import java.io.IOException;

public class FormatterPrimjer {
    public static void main(String[] args) {
        try (Formatter output = new Formatter("clients.txt")) {
            // Zapisivanje formatiranih podataka
            output.format("%d %s %s %.2f%n", 
                         1, "Ivan", "Horvat", 1500.50);
            output.format("%d %s %s %.2f%n", 
                         2, "Ana", "Kovač", 2300.75);
            output.format("%d %s %s %.2f%n", 
                         3, "Marko", "Babić", 1850.00);
            
            System.out.println("Podaci uspješno zapisani!");
            
        } catch (IOException | SecurityException e) {
            System.err.println("Pogreška: " + e.getMessage());
        }
    }
}
  • U slučaju nedostatka prava za korištenje datoteke baca se iznimka SecurityException

Čitanje i zapisivanje u binarne datoteke

Osnovna metoda klase InputStream

public abstract int read() throws IOException
  • Čita 1 bajt iz ulaznog toka
  • Vraća cjelobrojnu vrijednost tog bajta
  • Metoda read() blokira izvođenje ostatka programa dok se bajt ne pročita

Osnovna metoda klase OutputStream

public abstract void write(int b) throws IOException
  • Šalje 1 bajt podataka preko izlaznog toka
  • Odredište taj znak interpretira na određeni način
// Primjer kopiranja datoteke bajt po bajt
try (FileInputStream fis = new FileInputStream("ulaz.bin");
     FileOutputStream fos = new FileOutputStream("izlaz.bin")) {
    
    int bajt;
    while ((bajt = fis.read()) != -1) {
        fos.write(bajt);
    }
} catch (IOException e) {
    e.printStackTrace();
}

Blok "try-with-resources"

  • Svaku datoteku je nakon korištenja potrebno zatvoriti pozivom metode close()
  • Od Jave 7 uveden je poseban "try" blok koji automatski poziva metodu close()
  • Naziva se "try-with-resources"
import java.io.FileInputStream;
import java.io.IOException;

public class TryWithResourcesPrimjer {
    private static final String FILENAME = "test.txt";
    
    public static void main(String[] args) {
        // Objekt se automatski zatvara na kraju try bloka
        try (FileInputStream fis = new FileInputStream(FILENAME)) {
            
            for (int n = fis.read(); n != -1; n = fis.read()) {
                System.out.write(n);
            }
            System.out.flush();
            
        } catch (IOException ex) {
            System.err.println("Pogreška kod čitanja datoteke " + FILENAME);
            ex.printStackTrace();
        }
        // Nije potrebno ručno zvati fis.close()!
    }
}
✨ Prednosti try-with-resources
Automatsko zatvaranje resursa, čišći kod, manje šanse za memory leakove!

Unapređenja bloka "try-with-resources" u Javi 9

U Javi 9 je moguće koristiti blok "try-with-resources" na način da se objekti mogu definirati prije bloka i samo se koristiti u njemu:

Uvjet: objekti moraju biti "final" ili "effectively final"

import java.io.FileInputStream;
import java.io.IOException;

public class Java9TryWithResources {
    private static final String FILENAME = "test.txt";
    
    public static void main(String[] args) {
        // Objekt se kreira PRIJE try bloka
        FileInputStream fis = new FileInputStream(FILENAME);
        
        // Objekt se samo koristi u try bloku (mora biti effectively final)
        try (fis) {
            
            for (int n = fis.read(); n != -1; n = fis.read()) {
                System.out.write(n);
            }
            System.out.flush();
            
        } catch (IOException ex) {
            System.err.println("Pogreška kod čitanja datoteke " + FILENAME);
            ex.printStackTrace();
        }
        // fis se automatski zatvara!
    }
}
🎯 Effectively final
Varijabla je "effectively final" ako joj se vrijednost ne mijenja nakon inicijalizacije, čak i ako nije eksplicitno označena kao final.

Korištenje streamova u radu s datotekama

Klasa Files sadrži nekoliko metoda koje su vezane uz streamove i drastično olakšavaju rad s datotekama:

Metoda Opis
lines(Path putanja) Dohvaća sve linije unutar datoteke
list(Path putanja) Dohvaća sve datoteke na zadanoj putanji
newDirectoryStream(Path) Otvara mapu i omogućava dohvaćanje sadržaja
walk(Path putanja) Otvara mape i sve podmape u hijerarhiji
readString(Path) Čita cijeli sadržaj datoteke u jedan String
writeString(Path, String) Zapisuje cijeli String u datoteku 
import java.nio.file.*;
import java.io.IOException;

// Čitanje svih linija i filtriranje
Files.lines(Paths.get("podaci.txt"))
     .filter(line -> line.contains("Java"))
     .forEach(System.out::println);

// Ispis svih .java datoteka u mapi
Files.list(Paths.get("./src"))
     .filter(path -> path.toString().endsWith(".java"))
     .forEach(System.out::println);

Nakon dohvaćanja podataka, moguće je koristiti metode temeljene na lambda izrazima kao što su filter, forEach, limit, map itd.

Serijalizacija objekata u Javi

Za serijaliziranje objekata u binarnom obliku koristi se klasa ObjectOutputStream i metoda writeObject():

import java.io.*;
import java.util.Date;

public class SerijalizacijaPrimjer {
    public static void main(String[] args) {
        try (ObjectOutputStream out = new ObjectOutputStream(
                new FileOutputStream("osobe.dat"))) {
            
            // Kreiranje objekata
            Zupanija zagrebacka = new Zupanija(
                "Zagrebačka", 
                309696, 
                3078.0,
                Zupanija.POZIVNI_BROJ_ZUPANIJA_ZAGREBACKA_GRAD_ZAGREB
            );
            
            Osoba osoba = new Osoba(
                "Pero", 
                "Perić", 
                zagrebacka, 
                new Date()
            );
            
            // Serijalizacija objekta
            out.writeObject(osoba);
            
            System.out.println("Objekt uspješno serijaliziran!");
            
        } catch (IOException e) {
            System.err.println("Pogreška prilikom serijalizacije: " + e.getMessage());
            e.printStackTrace();
        }
    }
}
💡 Savjet
Možete serijalizirati više objekata u istu datoteku pozivajući writeObject() više puta.

Deserijalizacija objekata u Javi

Proces deserijalizacije obavlja se istim redoslijedom kojim su objekti i serijalizirani:

import java.io.*;

public class DeserijalizacijaPrimjer {
    private static final String SERIALIZATION_FILE_NAME = "osobe.dat";
    
    public static void main(String[] args) {
        try (ObjectInputStream in = new ObjectInputStream(
                new FileInputStream(SERIALIZATION_FILE_NAME))) {
            
            // Deserijalizacija objekta
            Osoba procitanaOsoba = (Osoba) in.readObject();
            
            // Ispis podataka o pročitanom objektu
            System.out.println("Podaci o pročitanom objektu:");
            System.out.println("Ime osobe: " + procitanaOsoba.getIme());
            System.out.println("Prezime osobe: " + procitanaOsoba.getPrezime());
            System.out.println("Županija: " + 
                procitanaOsoba.getZupanija().getNaziv());
            System.out.println("Datum rođenja: " + 
                procitanaOsoba.getDatumRodenja());
            
        } catch (IOException ex) {
            System.err.println("Pogreška kod čitanja datoteke: " + ex);
        } catch (ClassNotFoundException ex) {
            System.err.println("Klasa nije pronađena: " + ex);
        }
    }
}
⚠️ Važno
Potrebno je hvatati dvije iznimke: IOException i ClassNotFoundException

Zapisivanje i čitanje objekata u XML formatu

XML (eXtensible Markup Language) je fleksibilan način strukturiranja podataka u čitljivom formatu.

📋 Prednosti XML-a:

  • Čitljiv i razumljiv format za ljude
  • Platform-independent (ne ovisi o platformi)
  • Široko podržan u različitim programskim jezicima
  • Pogodan za konfiguracije i razmjenu podataka između sistema

🔧 Java XML API-ji:

JAXB (do Java 10)

  • Java Architecture for XML Binding
  • Automatsko mapiranje između Java objekata i XML-a
  • Od Java 11 je eksterena biblioteka

Jakarta XML Binding (Java 11+)

  • Nasljednik JAXB-a
  • Dio Jakarta EE
  • Preporučen za moderne aplikacije
💡 Zanimljivost
XML se koristi u mnogim poznatim tehnologijama: Android manifesti, Maven pom.xml, Spring konfiguracije, RSS feedovi itd.

Zapisivanje objekata u XML datoteku

import jakarta.xml.bind.*;
import java.io.File;

public class XMLSerijalizacija {
    public static void main(String[] args) {
        try {
            // Kreiranje objekta studenta
            Student student = new Student(
                1, 
                "Ana", 
                "Horvat", 
                4.8
            );
            
            // Kreiranje JAXB konteksta
            JAXBContext context = JAXBContext.newInstance(Student.class);
            
            // Kreiranje marshallera (za zapisivanje u XML)
            Marshaller marshaller = context.createMarshaller();
            
            // Postavljanje formatiranja (za ljepši ispis)
            marshaller.setProperty(Marshaller.JAXB_FORMATTED_OUTPUT, true);
            marshaller.setProperty(Marshaller.JAXB_ENCODING, "UTF-8");
            
            // Zapisivanje u datoteku
            File xmlFile = new File("student.xml");
            marshaller.marshal(student, xmlFile);
            
            // Ispis na ekran za provjeru
            marshaller.marshal(student, System.out);
            
            System.out.println("\nStudent uspješno zapisan u XML!");
            
        } catch (JAXBException e) {
            System.err.println("Pogreška kod XML serijalizacije: " 
                             + e.getMessage());
            e.printStackTrace();
        }
    }
}

Čitanje objekata iz XML datoteke

import jakarta.xml.bind.*;
import java.io.File;

public class XMLDeserijalizacija {
    public static void main(String[] args) {
        try {
            // Kreiranje JAXB konteksta
            JAXBContext context = JAXBContext.newInstance(Student.class);
            
            // Kreiranje unmarshallera (za čitanje iz XML-a)
            Unmarshaller unmarshaller = context.createUnmarshaller();
            
            // Čitanje objekta iz datoteke
            File xmlFile = new File("student.xml");
            Student student = (Student) unmarshaller.unmarshal(xmlFile);
            
            // Ispis učitanih podataka
            System.out.println("Podaci o pročitanom studentu:");
            System.out.println("ID: " + student.getId());
            System.out.println("Ime: " + student.getIme());
            System.out.println("Prezime: " + student.getPrezime());
            System.out.println("Prosjek: " + student.getProsjek());
            
        } catch (JAXBException e) {
            System.err.println("Pogreška kod XML deserijalizacije: " 
                             + e.getMessage());
            e.printStackTrace();
        }
    }
}

Zapisivanje i čitanje objekata u JSON formatu

JSON (JavaScript Object Notation) je lagan, čitljiv format za razmjenu podataka koji je postao standard u modernom razvoju softvera.

🚀 Prednosti JSON-a:

  • Jednostavniji i komaktniji od XML-a
  • Native format za JavaScript i web aplikacije
  • Brža obrada i manja veličina datoteka
  • Dominantan format u REST API-jima
  • Odličan za moderne microservice arhitekture

🔧 Java JSON biblioteke:

Jakarta JSON Binding (JSON-B)

  • Službeni Java EE/Jakarta EE standard
  • Sličan JAXB-u, ali za JSON
  • Jednostavan za korištenje

Jackson

  • Najpopularnija JSON biblioteka
  • Visoke performanse
  • Bogat feature set
💡 Zanimljivost za Z generaciju
Svaki put kad koristite Instagram, TikTok, Spotify ili bilo koju drugu mobilnu app - šaljete i primate podatke u JSON formatu! 📱

Zapisivanje objekata u JSON datoteku

import jakarta.json.bind.*;
import java.io.*;
import java.util.*;

public class JSONSerijalizacija {
    public static void main(String[] args) {
        try {
            // Kreiranje Jsonb instance
            Jsonb jsonb = JsonbBuilder.create();
            
            // Kreiranje jednog korisnika
            Korisnik korisnik = new Korisnik(
                1, 
                "ana.horvat", 
                "[email protected]",
                true
            );
            
            // Zapisivanje objekta u JSON string
            String json = jsonb.toJson(korisnik);
            System.out.println("JSON reprezentacija:\n" + json);
            
            // Zapisivanje u datoteku
            try (FileWriter writer = new FileWriter("korisnik.json")) {
                writer.write(json);
            }
            
            // Možemo raditi i s listama objekata!
            List<Korisnik> korisnici = Arrays.asList(
                new Korisnik(1, "ana.horvat", "[email protected]", true),
                new Korisnik(2, "marko.markic", "[email protected]", true),
                new Korisnik(3, "ivana.ivic", "[email protected]", false)
            );
            
            String jsonLista = jsonb.toJson(korisnici);
            try (FileWriter writer = new FileWriter("korisnici.json")) {
                writer.write(jsonLista);
            }
            
            System.out.println("\n✅ Podaci uspješno zapisani u JSON!");
            
        } catch (Exception e) {

Čitanje objekata iz JSON datoteke

import jakarta.json.bind.*;
import java.io.*;
import java.nio.file.*;
import java.util.*;

public class JSONDeserijalizacija {
    public static void main(String[] args) {
        try {
            // Kreiranje Jsonb instance
            Jsonb jsonb = JsonbBuilder.create();
            
            // Čitanje JSON-a iz datoteke
            String json = Files.readString(Paths.get("korisnik.json"));
            
            // Deserijalizacija u objekt
            Korisnik korisnik = jsonb.fromJson(json, Korisnik.class);
            
            // Ispis podataka
            System.out.println("Pročitani podaci o korisniku:");
            System.out.println("ID: " + korisnik.getId());
            System.out.println("Korisničko ime: " + korisnik.getKorisnickoIme());
            System.out.println("Email: " + korisnik.getEmail());
            System.out.println("Aktivan: " + korisnik.isAktivan());
            
            // Čitanje liste objekata
            String jsonLista = Files.readString(Paths.get("korisnici.json"));
            
            // Deserijalizacija liste - koristimo Type reference
            List<Korisnik> korisnici = jsonb.fromJson(
                jsonLista, 
                new ArrayList<Korisnik>(){}.getClass().getGenericSuperclass()
            );
            
            System.out.println("\nLista korisnika:");
            for (Korisnik k : korisnici) {
                System.out.printf("- %s (%s) - %s%n", 
                    k.getKorisnickoIme(), 
                    k.getEmail(),
                    k.isAktivan() ? "Aktivan" : "Neaktivan"
                );
            }
            
        } catch (Exception e) {
            System.err.println("Pogreška: " + e.getMessage());
            e.printStackTrace();
        }
    }
}

Usporedba: Binary vs XML vs JSON

Karakteristika Binary (Serializable) XML JSON
Čitljivost ❌ Nije čitljiv ljudima ✅ Čitljiv ✅ Vrlo čitljiv
Veličina datoteke ✅ Najmanja ❌ Najveća ✅ Mala do srednja
Brzina ✅ Najbrža ❌ Sporija ✅ Brza
Interoperabilnost ❌ Samo Java ✅ Svi jezici ✅ Svi jezici
Validacija schema ❌ Nema ✅ XSD ✅ JSON Schema
Web API ❌ Ne koristi se ⚠️ SOAP (legacy) ✅ REST (standard)

📊 Kada koristiti koji format?

✅ Koristi Binary serijalizaciju
  • Interni Java sustavi
  • Cache mehanizmi
  • Performanse su kritične
✅ Koristi XML
  • Konfiguracijske datoteke
  • Legacy sistemi
  • Potrebna je validacija schema
✅ Koristi JSON (najpopularniji izbor!)
  • REST API-ji
  • Web aplikacije
  • Mobile aplikacije
  • Microservices
  • NoSQL baze (MongoDB)
  • Config datoteke (moderne)

Praktični primjer: REST API s JSON-om

Primjer kako se JSON koristi u stvarnoj web aplikaciji s Spring Boot-om:

import org.springframework.web.bind.annotation.*;
import java.util.*;

@RestController
@RequestMapping("/api/korisnici")
public class KorisnikController {
    
    private List<Korisnik> korisnici = new ArrayList<>();
    
    // GET zahtjev - vraća JSON
    @GetMapping
    public List<Korisnik> dohvatiSveKorisnike() {
        return korisnici;  // Automatski se pretvara u JSON!
    }
    
    // GET zahtjev s parametrom - vraća jedan objekt kao JSON
    @GetMapping("/{id}")
    public Korisnik dohvatiKorisnika(@PathVariable int id) {
        return korisnici.stream()
            .filter(k -> k.getId() == id)
            .findFirst()
            .orElse(null);
    }
    
    // POST zahtjev - prima JSON i kreira objekt
    @PostMapping
    public Korisnik kreirajKorisnika(@RequestBody Korisnik korisnik) {
        korisnici.add(korisnik);
        return korisnik;  // Vraća JSON natrag klijentu
    }
    
    // PUT zahtjev - ažurira postojeći objekt
    @PutMapping("/{id}")
    public Korisnik azurirajKorisnika(@PathVariable int id, 
                                       @RequestBody Korisnik noviPodaci) {
        Korisnik postojeci = korisnici.stream()
            .filter(k -> k.getId() == id)
            .findFirst()
            .orElse(null);
            
        if (postojeci != null) {
            postojeci.setKorisnickoIme(noviPodaci.getKorisnickoIme());
            postojeci.setEmail(noviPodaci.getEmail());
            postojeci.setAktivan(noviPodaci.isAktivan());
        }
        return postojeci;
    }
}
💡 Što se događa u pozadini?
Spring Boot automatski koristi Jackson biblioteku za konverziju između JSON-a i Java objekata. Programer ne mora ručno pozivati JsonbBuilder!

Best Practices za rad s datotekama

1️⃣ Uvijek koristi try-with-resources
Automatski zatvara resurse i sprječava memory leakove
2️⃣ Koristi NIO.2 API (java.nio.file)
Moderniji, brži i fleksibilniji od starog java.io
3️⃣ Koristi buffering za velike datoteke
BufferedReader/Writer drastično poboljšavaju performanse
4️⃣ Hvtaj specifične iznimke
IOException, FileNotFoundException, itd.
5️⃣ JSON > XML za nove projekte
Jednostavniji, brži, bolji za web i mobile
6️⃣ Postavi encoding eksplicitno
Koristi UTF-8 za compatibility
7️⃣ Validiraj putanje datoteka
Provjeri da datoteka postoji prije čitanja
8️⃣ Ne čitaj cijelu veliku datoteku u memoriju
Koristi streamove za procesiranje liniju po liniju
9️⃣ Koristi Files utility metode
Files.lines(), Files.readString(), Files.writeString()
🔟 Dokumentiraj format datoteka
Jasno napiši što tvoj program očekuje
