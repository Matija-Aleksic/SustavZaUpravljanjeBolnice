🎯 Što su to iznimke?

Iznimka (Exception) je objekt koji predstavlja problem nastao tijekom izvođenja programa.

  • 🛡️ Omogućavaju razvoj robusnih aplikacija
  • ♻️ Program može nastaviti rad i nakon greške
  • 🎭 Predstavljene klasama koje nasljeđuju Throwable
  • 🎣 Mogu se hvatati (catch) i bacati (throw)
Exception handling nije samo za greške - koristi se i za kontrolu toka programa!

❌ Problem: Bez obrade iznimki

public class BezObradeIznimke {
    private static int podijeli(int djeljenik, int djelitelj) {
        return djeljenik / djelitelj;  // ⚠️ Što ako je djelitelj 0?
    }
    
    public static void main(String[] args) {
        Scanner unos = new Scanner(System.in);
        System.out.print("Unesite dva broja: ");
        
        int prvi = unos.nextInt();
        int drugi = unos.nextInt();
        int rezultat = podijeli(prvi, drugi);
        
        System.out.println("Rezultat: " + rezultat);
        unos.close();
    }
}

✅ Rješenje: Try-Catch blok

boolean nastaviPetlju;
do {
    nastaviPetlju = false;
    try {
        // Rizičan kod koji može baciti iznimku
        System.out.print("Unesite dva broja: ");
        int prvi = unos.nextInt();
        int drugi = unos.nextInt();
        int rezultat = podijeli(prvi, drugi);
        System.out.println("Rezultat: " + rezultat);
        
    } catch (InputMismatchException ex) {
        // Obrada pogrešnog unosa
        System.out.println("❌ Morate unijeti brojčane vrijednosti!");
        unos.nextLine();  // Očisti buffer
        nastaviPetlju = true;
        
    } catch (ArithmeticException ex) {
        // Obrada dijeljenja s nulom
        System.out.println("❌ Ne možete dijeliti s nulom!");
        unos.nextLine();
        nastaviPetlju = true;
    }
} while (nastaviPetlju);

🔬 Anatomija Try-Catch bloka

TRY blok

Sadrži kod koji može baciti iznimku. Izvodi se sve dok ne dođe do iznimke.

CATCH blok

Sadrži kod koji se izvodi samo ako se dogodi određena iznimka.

Možete imati više catch blokova za različite tipove iznimaka!
  • ⚡ Try se izvodi linearno do iznimke
  • 🎯 Catch "hvata" samo svoj tip iznimke
  • 🔄 Nakon catch-a program normalno nastavlja

🎭 Multi-Catch blok Java 7+

Kada više iznimaka treba obrađivati na isti način:

❌ Stari način

catch (IOException ex) {
    logger.log(ex);
    showError();
}
catch (SQLException ex) {
    logger.log(ex);
    showError();
}

✅ Novi način

catch (IOException | SQLException ex) {
    logger.log(ex);
    showError();
}
Prednosti:
  • Manje dupliciranog koda
  • Čitljiviji i koncizniji kod
  • Lakše održavanje

🎬 Finally Blok - Uvijek se izvršava!

FileReader reader = null;
try {
    reader = new FileReader("data.txt");
    // Rad s datotekom...
    
} catch (IOException ex) {
    System.err.println("Greška čitanja: " + ex.getMessage());
    
} finally {
    // Ovo se UVIJEK izvršava - iznimka ili ne!
    if (reader != null) {
        try {
            reader.close();
            System.out.println("✅ Datoteka zatvorena");
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
Finally se izvršava: Nakon try bloka, nakon catch bloka, čak i ako postoji return u try/catch!

🚰 Try-with-Resources Java 7+

Automatsko zatvaranje resursa - elegantnije i sigurnije!

❌ Stari način (verbozno)

BufferedReader br = null;
try {
    br = new BufferedReader(
        new FileReader("file.txt")
    );
    return br.readLine();
} finally {
    if (br != null) {
        br.close();
    }
}

✅ Novi način (clean)

try (BufferedReader br = 
        new BufferedReader(
            new FileReader("file.txt")
        )) {
    return br.readLine();
}
// Automatski zatvoreno! 🎉
Radi s bilo kojom klasom koja implementira AutoCloseable interface!

🌳 Hijerarhija iznimaka

java.lang.Throwable - Korijen svega
Exception - Iznimke koje treba obrađivati
RuntimeException - Neoznačene iznimke
ArithmeticException NullPointerException IndexOutOfBoundsException
IOException - Označene iznimke
FileNotFoundException EOFException
Error - Ozbiljne greške (NE OBRAĐIVATI!)
OutOfMemoryError StackOverflowError

🏷️ Označene vs Neoznačene iznimke

✅ Checked (Označene)

  • Nasljeđuju Exception
  • MORAJU se obrađivati
  • Kompajler ih provjerava
  • Za predvidive probleme
IOException SQLException ClassNotFoundException

⚠️ Unchecked (Neoznačene)

  • Nasljeđuju RuntimeException
  • Ne moraju se obrađivati
  • Kompajler ih ne provjerava
  • Za programske greške
NullPointerException ArithmeticException ArrayIndexOutOfBoundsException
Pravilo: Ako možete spriječiti u kodu - unchecked. Ako ovisi o vanjskim faktorima - checked.

🎾 Bacanje iznimaka - throw

public class Korisnik {
    private String email;
    
    public void setEmail(String email) {
        // Validacija emaila
        if (email == null || email.isEmpty()) {
            throw new IllegalArgumentException(
                "Email ne može biti prazan! 📧"
            );
        }
        
        if (!email.contains("@")) {
            throw new IllegalArgumentException(
                "Email mora sadržavati @"
            );
        }
        
        this.email = email;
    }
}
throw - baca novu iznimku (stvara objekt i prekida tok izvođenja)

📢 Deklaracija iznimaka - throws

Za checked iznimke morate ili: (1) uhvatiti ih ili (2) deklarirati da ih metoda baca

public class FileManager {
    
    // Metoda deklarira da baca IOException
    public String readFile(String path) 
            throws IOException {
        
        BufferedReader reader = new BufferedReader(
            new FileReader(path)  // Baca IOException
        );
        
        return reader.readLine();  // Također baca IOException
    }
    
    public void processFile() {
        try {
            String content = readFile("data.txt");
            System.out.println(content);
        } catch (IOException e) {
            System.err.println("❌ Greška: " + e.getMessage());
        }
    }
}
throws = "ova metoda može baciti ove iznimke, pa ih ti obradi"

🎨 Kreiranje vlastitih iznimaka

// Checked iznimka (nasljeđuje Exception)
public class InvalidEmailException extends Exception {
    
    public InvalidEmailException() {
        super("Email adresa nije valjana");
    }
    
    public InvalidEmailException(String message) {
        super(message);
    }
    
    public InvalidEmailException(String message, Throwable cause) {
        super(message, cause);
    }
    
    public InvalidEmailException(Throwable cause) {
        super(cause);
    }
}
Best Practice: Implementirajte sva 4 konstruktora za fleksibilnost!

🎯 Korištenje vlastite iznimke

public class EmailValidator {
    
    public static void validateEmail(String email) 
            throws InvalidEmailException {
        
        if (email == null || email.isEmpty()) {
            throw new InvalidEmailException(
                "Email ne smije biti prazan"
            );
        }
        
        if (!email.matches("^[A-Za-z0-9+_.-]+@(.+)$")) {
            throw new InvalidEmailException(
                "Email format nije valjan: " + email
            );
        }
    }
    
    public static void main(String[] args) {
        try {
            validateEmail("korisnik@primjer.com");
            System.out.println("✅ Email je valjan!");
            
        } catch (InvalidEmailException e) {
            System.err.println("❌ " + e.getMessage());
        }
    }
}

⭐ Best Practices

✅ RADI:

  • Hvati specifične iznimke, ne Exception
  • Daj korisne poruke o greškama
  • Koristi try-with-resources za resurse
  • Logiraj iznimke za debugging
  • Kreiraj vlastite iznimke za domensku logiku

❌ NEMOJ:

  • Prazan catch blok (catch(Exception e) {})
  • Hvatati Error klasu
  • Koristiti iznimke za kontrolu toka
  • Ignorirati checked iznimke
  • Bacati generičke iznimke bez poruke

🆕 Pattern Matching Java 21+

Moderne mogućnosti za obradu iznimaka:

public String handleException(Exception ex) {
    return switch (ex) {
        case FileNotFoundException fnf 
            when fnf.getMessage().contains("config") ->
            "⚙️ Konfiguracijska datoteka nije pronađena";
            
        case FileNotFoundException fnf ->
            "📄 Datoteka nije pronađena: " + fnf.getMessage();
            
        case IOException io ->
            "💾 I/O greška: " + io.getMessage();
            
        case SQLException sql ->
            "🗃️ Greška baze podataka: " + sql.getMessage();
            
        default ->
            "❌ Nepoznata greška: " + ex.getMessage();
    };
}
Pattern matching omogućava elegantniju obradu različitih tipova iznimaka!

⛓️ Chained Exceptions

Povezivanje iznimaka za bolju dijagnostiku:

public class DataService {
    
    public User getUser(int id) throws DataAccessException {
        try {
            // Spajanje na bazu i dohvaćanje korisnika
            Connection conn = getConnection();
            return fetchUserFromDB(conn, id);
            
        } catch (SQLException e) {
            // Omotaj SQL iznimku u domensku iznimku
            throw new DataAccessException(
                "Ne mogu dohvatiti korisnika #" + id,
                e  // ← Originalna iznimka kao uzrok
            );
        }
    }
}
Prednosti:
  • Zadržava potpunu informaciju o greški
  • Omogućava apstrakciju (skriva SQL detalje)
  • Olakšava debugging (cijeli lanac je vidljiv)

💼 Praktični primjer - API poziv

public class UserAPI {
    
    public User createUser(String email, String password) {
        try {
            // Validacija
            EmailValidator.validateEmail(email);
            PasswordValidator.validatePassword(password);
            
            // Provjera postoji li već korisnik
            if (userRepository.existsByEmail(email)) {
                throw new UserAlreadyExistsException(email);
            }
            
            // Kreiranje korisnika
            User user = new User(email, password);
            return userRepository.save(user);
            
        } catch (InvalidEmailException | InvalidPasswordException e) {
            // Validacijske greške
            logger.warn("Validacija neuspješna: {}", e.getMessage());
            throw new ValidationException(e.getMessage(), e);
            
        } catch (UserAlreadyExistsException e) {
            // Poslovni problem
            logger.info("Pokušaj duple registracije: {}", email);
            throw e;
            
        } catch (DataAccessException e) {
            // Tehnički problem
            logger.error("DB greška pri kreiranju korisnika", e);
            throw new ServiceException("Nije moguće kreirati korisnika", e);
        }
    }
}
