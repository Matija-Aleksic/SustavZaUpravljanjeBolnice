🖥️ Grafička sučelja u Javi

GUI (Graphical User Interface) omogućava korištenje aplikacija kroz grafičke elemente - kontrole ili widgete (Window Gadgets)

  • AWT (Abstract Window Toolkit) - prvi GUI library u Javi, prisutan od samih početaka
  • Swing - primarni GUI toolkit od Jave SE 1.2 do Jave SE 7
  • Swing je trenutno samo u fazi održavanja (maintenance mode)
  • Ostaje dio Jave radi backward compatibility, ali se više ne razvija aktivno

📜 Povijest JavaFX-a

✨ Prednosti JavaFX-a

🎨 Scene Builder

Vizualni alat za dizajniranje GUI-ja. Može se koristiti samostalno ili integriran u IDE.

💅 CSS Styling

Korištenje CSS-a za oblikovanje komponenti - poznato web developerima!

⚡ Multithreading

Napredno korištenje niti za optimalne performanse na višejezgrenim procesorima.

🎬 Animacije

Podrška za transformacije i animacije - dinamični UI efekti.

👨‍💻 Prva JavaFX aplikacija

Main.java 
public class Main extends Application {
    @Override
    public void start(Stage primaryStage) {
        try {
            BorderPane root = new BorderPane();
            Scene scene = new Scene(root, 400, 400);
            scene.getStylesheets().add(
                getClass().getResource("application.css")
                    .toExternalForm()
            );
            primaryStage.setScene(scene);
            primaryStage.show();
        } catch(Exception e) {
            e.printStackTrace();
        }
    }

    public static void main(String[] args) {
        launch(args);
    }
}
JavaFX Application

📱 Rezultat: prazan prozor 400x400

🔍 Razumijevanje strukture

  • extends Application - nasljeđivanje klase koja upravlja životnim ciklusom aplikacije
  • start(Stage primaryStage) - glavna metoda za pokretanje GUI-ja, prima Stage objekt
  • Stage - glavni prozor aplikacije (kao pozornica u kazalištu 🎭)
  • Scene - sadržaj prozora, može se mijenjati dinamički
  • launch(args) - pokreće JavaFX aplikaciju iz main metode

💡 Scene Graph - JavaFX koncept koji omogućuje korištenje više Scene objekata na jednom Stage objektu

💅 CSS stiliziranje u JavaFX-u

Dodavanje CSS datoteke:

scene.getStylesheets().add(
    getClass().getResource("application.css")
        .toExternalForm()
);

Inline stil u kodu:

root.setStyle(
    "-fx-background: rgb(225, 228, 203)"
);

⚠️ Važno: CSS svojstva u JavaFX-u moraju imati -fx- prefiks!

application.css 
.root {
    -fx-font-size: 16pt;
    -fx-font-family: "Courier New";
    -fx-background: rgb(225, 228, 203);
}

📄 FXML - FX Markup Language

Scene Builder generira FXML datoteke - XML format za definiranje GUI-ja

  • Odvaja prezentacijsku logiku od programske logike
  • Lakše održavanje i promjene dizajna bez diranja Java koda
// Učitavanje FXML datoteke
BorderPane root = (BorderPane) FXMLLoader.load(
    getClass().getResource("Sample.fxml")
);

📱 Forma u GridPane-u

JavaFX MVC

Model - View - Controller arhitektura

🏗️ Osnove MVC arhitekture

📦 Model

Klase koje predstavljaju domenske podatke (npr. Knjiga, Korisnik, Proizvod)

👁️ View

FXML datoteke ili Java klase koje definiraju izgled grafičkog sučelja

🎮 Controller

Povezuje GUI elemente s poslovnom logikom - reagira na korisničke akcije

💡 Ključno: Elementi GUI-ja trebaju imati jedinstvene identifikatore (fx:id) za povezivanje s Controllerom

🎮 Controller klasa

  • Povezuje elemente GUI-ja s Java kodom (poslovna logika)
  • Varijable moraju odgovarati elementima GUI-ja po tipu i nazivu
  • Svaki ekran trebao bi imati svoju FXML datoteku i Controller klasu
Sample.fxml 
<BorderPane fx:controller="hr.tvz.controller.KnjigeController">
    ...
</BorderPane>

🏷️ Anotacija @FXML

Označava elemente u Javi koji se koriste u FXML-u:

KnjigeController.java 
public class KnjigeController {

    @FXML
    private TableView<Knjiga> knjigaTable;
    
    @FXML
    public void initialize() {
        // inicijalizacija ekrana
    }
    
    @FXML
    public void prikaziSveKnjige() {
        // logika za prikaz
    }
}

⚠️ Važno: Naziv varijable mora odgovarati fx:id u FXML-u!

knjige.fxml 
<TableView fx:id="knjigaTable">
    ...
</TableView>

<Button text="Prikaži"
    onAction="#prikaziSveKnjige"/>

📚 JavaFX zbirke (Collections)

Specijalne zbirke koje automatski ažuriraju GUI pri promjenama:

ObservableList

Lista s automatskim praćenjem promjena preko Listenera

ObservableMap

Mapa s automatskim praćenjem promjena

FXCollections

Factory klasa za kreiranje Observable zbirki

ListChangeListener

Prima notifikacije o promjenama u listi

📋 ObservableList - Primjer

// Kreiranje ObservableList iz obične Java liste
List<Knjiga> filtriraneKnjige = dohvatiKnjige();

ObservableList<Knjiga> listaKnjiga = 
    FXCollections.observableArrayList(filtriraneKnjige);

// Povezivanje s TableView komponentom
knjigaTable.setItems(listaKnjiga);

// Sada kada se promijeni listaKnjiga, 
// TableView se automatski ažurira! 🎉

💡 Ovo je kao reactive programming - promjena podataka automatski ažurira prikaz, slično kao u React-u ili Vue.js!

🔗 Povezivanje stupaca s podacima

@FXML
private TableColumn<Knjiga, String> nazivKnjigeColumn;

@FXML
public void initialize() {
    nazivKnjigeColumn.setCellValueFactory(
        new Callback<CellDataFeatures<Knjiga,String>, 
                     ObservableValue<String>>() {
            public ObservableValue<String> call(
                    CellDataFeatures<Knjiga, String> param) {
                return new ReadOnlyStringWrapper(
                    param.getValue().getNaziv()
                );
            }
        }
    );
}

📑 Povezivanje akcija s izbornikom

Izbornik se sastoji od Menu (glavni element) i MenuItem elemenata:

public void prikaziKnjige() {
    try {
        BorderPane knjigaPane = FXMLLoader.load(
            JavaFXGlavna.class.getResource("../javafx/knjige.fxml")
        );
        JavaFXGlavna.setCenterPane(knjigaPane);
    } catch (IOException e) {
        e.printStackTrace();
    }
}

💡 Best practice: Svaka opcija izbornika otvara novi ekran definiran u zasebnoj FXML datoteci
